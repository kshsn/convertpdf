# i18n Rollout — Handoff Brief

Goal: make ConvertPDF rank/be-found in 8 languages. This brief lets a fresh session
finish the rollout safely on the **live, indexed, monetized** site.

## Decisions already made (do not re-litigate)
- **Locales (8):** en, ar, es, fr, pt, de, hi, id. English is default.
- **URL strategy:** English stays at root (`/merge-pdf` — unchanged, already indexed).
  Other locales are prefixed (`/ar/merge-pdf`, `/es/merge-pdf`, …). This protects existing
  SEO — do NOT move English under `/en`.
- Translations are generated by the assistant (good quality for this short content).

## Already done & committed (foundation — safe, nothing wired in yet)
- `lib/i18n/config.ts` — locales, defaultLocale, rtlLocales (ar), localeNames, helpers.
- `lib/i18n/dictionaries.ts` — `Dictionary` type + `getDictionary(locale)` loader
  (server-only; falls back to English for any missing locale).
- `lib/i18n/locales/en.json` — full English dictionary (nav, footer, home, common UI,
  all 8 tools: title/tagline/intro/faq).
- `lib/i18n/locales/ar.json` — full Arabic dictionary (RTL).

## Remaining work
### 1. Dictionaries (mechanical — copy en.json shape, translate values)
Create `lib/i18n/locales/{es,fr,pt,de,hi,id}.json` with the same keys as en.json.
The loader already references them. Tool slugs: merge-pdf, split-pdf, compress-pdf,
pdf-to-word, word-to-pdf, rotate-pdf, pdf-to-jpg, protect-pdf.

### 2. Routing (the careful part — commit + `npm run build` after each step)
- `middleware.ts`: detect locale. If path has no locale prefix AND it's not a non-default
  locale, serve English (root). For `/ar`, `/es`, … route to `app/[locale]`. Optionally
  redirect first-time visitors by `Accept-Language` (use a cookie to avoid redirect loops;
  do NOT auto-redirect crawlers off the indexed English root — keep English crawlable at root).
- Refactor home + tool pages into **dictionary-driven shared components** so English-at-root
  and `/[locale]` reuse the same code (avoid duplicating bespoke tool logic: merge multi-file,
  split range, rotate angle, protect password).
- `app/[locale]/layout.tsx`: validate locale (404 if not in `locales`), set `<html lang dir>`
  (RTL for ar), `generateStaticParams` for the 7 non-default locales, hreflang via
  `alternates.languages` in metadata.
- Add **hreflang alternates** to every page (all 8 locales + x-default) — both root English
  pages and `/[locale]` pages must cross-reference each other.
- **Language switcher** component (uses `localeNames`), placed in Navbar/Footer.
- Update `app/sitemap.ts` to emit all locale URLs with hreflang.

### 3. Verify
- `npm run build` passes; visit `/`, `/ar` (RTL), `/ar/merge-pdf`, etc.
- View source: correct `<title>`/`<meta>` per language, `lang`/`dir` correct, hreflang present.
- English URLs unchanged (no redirects off `/merge-pdf`).

## Guardrails (live site)
- Never break existing English URLs — they're indexed. No 301s off root.
- Keep AdSense meta tag, `/ads.txt`, GA, canonical tags working on all pages.
- Commit one file per commit; `npm run build` before every push; deploy is
  `git pull && npm run build && pm2 restart convertpdf --update-env` on the VPS (port 3000).
